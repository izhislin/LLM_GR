# ADR

## Записи

### ADR-001: PyTorch nightly для RTX 5060 Ti (Blackwell)

**Дата:** 2026-03-12
**Статус:** Принято
**Контекст:** RTX 5060 Ti использует архитектуру Blackwell (sm_120). Стабильные релизы PyTorch (до 2.10.0 включительно) не поддерживают sm_120 — только до sm_90 (Hopper).
**Решение:** Используем PyTorch nightly `2.12.0.dev+cu128`. Wheel скачивается локально на Mac и передаётся на сервер по SCP, т.к. CDN PyTorch таймаутит с сервера.
**Риск:** Nightly-сборка может быть нестабильной. При появлении стабильного релиза с поддержкой sm_120 — обновить.
**Альтернативы:** Сборка PyTorch из исходников с `TORCH_CUDA_ARCH_LIST="sm_120"` — отклонена как избыточно сложная.

### ADR-002: GigaAM v3 из GitHub вместо PyPI

**Дата:** 2026-03-12
**Статус:** Принято
**Контекст:** PyPI-пакет `gigaam==0.1.0` содержит только модели v1/v2. Модель `v3_e2e_rnnt` доступна только в GitHub-репозитории `salute-developers/GigaAM`.
**Решение:** Устанавливаем из GitHub с `--no-deps` для обхода ограничения `torch<2.9`.
**Риск:** При обновлении GigaAM API может измениться. Проверять совместимость при обновлениях.
