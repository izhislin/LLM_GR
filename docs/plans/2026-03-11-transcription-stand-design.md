# Дизайн: Тестовый стенд транскрибации и LLM-обработки звонков

**Дата:** 2026-03-11
**Статус:** Утверждён

## Цель

Собрать локальный тестовый стенд для потоковой транскрибации двухканальных телефонных аудиозаписей и их интеллектуальной обработки (суммаризация, оценка качества, извлечение данных).

## Выбранный подход: GigaAM-v3 + Qwen3-8B

### ASR: GigaAM-v3 (e2e_rnnt)

- **Модель:** GigaAM-v3 от Сбера (SaluteSpeech), MIT лицензия
- **Параметры:** ~240M, ~450MB на диске
- **VRAM:** ~1 GB (fp16)
- **Качество:** WER 8.4% (средний по русским бенчмаркам), 9.5% на колл-центр данных
- **Превосходство над Whisper:** в 2-3 раза ниже WER на русском языке
- **Ограничение:** Требует предварительной сегментации длинного аудио (VAD/pyannote)

### LLM: Qwen3-8B (Q4_K_M через Ollama)

- **Модель:** Qwen3-8B-Instruct, Apache 2.0 лицензия
- **Квантизация:** Q4_K_M (~5GB файл)
- **VRAM:** ~6-8 GB
- **Качество:** Уровень Qwen2.5-14B, хорошая поддержка русского (119 языков)
- **Режимы:** Гибридный thinking/non-thinking для разных задач
- **Сервинг:** Ollama (REST API на localhost:11434)

### Бюджет VRAM

| Компонент | VRAM |
|---|---|
| GigaAM-v3 | ~1 GB |
| Qwen3-8B Q4_K_M | ~6-8 GB |
| **Итого** | **~7-9 GB из 16 GB** |
| Запас для экспериментов | ~7-9 GB |

## Пайплайн обработки

```
Аудиофайл (stereo WAV/MP3)
  → ffmpeg: разделение на 2 моно-канала (оператор / клиент)
  → GigaAM-v3: транскрипция каждого канала с таймкодами
  → Сборка хронологического диалога с метками говорящих
  → Qwen3-8B (вызов 1): суммаризация
  → Qwen3-8B (вызов 2): оценка качества
  → Qwen3-8B (вызов 3): извлечение данных
  → JSON-результат
```

## Стратегия промптов

- Три отдельных вызова LLM (суммаризация, оценка, извлечение) вместо одного мега-промпта
- Промпты хранятся в `prompts/*.md`, каждый содержит роль, формат, примеры
- При невалидном JSON от LLM — повторная попытка с упрощённым промптом

## Масштабирование (будущее)

Тестовый стенд → продакшн-сервер с более мощным GPU для 50+ одновременных звонков.
Архитектура спроектирована для горизонтального масштабирования: ASR и LLM можно разнести на разные GPU/серверы.

## Альтернативы (рассмотрены и отклонены)

- **Faster-Whisper** — отклонён из-за в 2-3 раза худшего WER на русском (особенно на телефонной речи)
- **Vikhr-Nemo-12B** — рассмотрен для LLM, оставлен как запасной вариант для экспериментов
- **vLLM** — избыточен для single-user тестового стенда
