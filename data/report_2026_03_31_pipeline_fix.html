3 Корневая причина

Конфликт между webhook и polling при обработке одних и тех же звонков:

Цепочка событий

26 марта — фикс webhook
Исправлен баг с суффиксом /event в URL. Webhook начал принимать звонки (ранее все шли 404).
27 марта — начало проблемы
Webhook создаёт запись звонка мгновенно с пустым record_url и duration=0 (запись ещё не готова). Фильтр ставит skipped: no record.
+10 мин — polling
Polling получает тот же звонок от API с записью и длительностью. Но get_call() находит существующий ID → continue → данные не обновляются.

Проблемный код

src/web/app.py, функция _poll_domain:

if not call_id or get_call(_db, call_id):
    continue  # polling пропускает уже известные звонки

INSERT OR IGNORE в insert_call также не обновляет существующие записи.

До 26 марта webhook-и не работали (404), поэтому все звонки приходили через polling с полными данными. После фикса webhook стал перехватывать звонки первым, создавая «пустые» записи.

4 Исправление

4.1 Новые функции в src/db.py

  • update_call_from_polling() — обновляет record_url, duration, result и другие поля, только если текущий record_url пуст
  • reopen_processing() — переводит запись из skipped → pending с очисткой skip_reason

4.2 Изменение _poll_domain

Вместо continue при существующем звонке — обновление данных и переоценка фильтра:

existing = get_call(_db, call_id)
if existing:
    if not existing["record_url"] and call["record_url"]:
        update_call_from_polling(_db, call_id, call)
        passed, _reason = filter_call(call, config.filters)
        if passed:
            reopen_processing(_db, call_id)
    continue

4.3 Бэкфилл при старте

Одноразовый проход по истории за последние 7 дней при запуске приложения, чтобы подтянуть record_url для ранее пропущенных звонков. В дальнейшем служит safety net при перезапусках.

4.4 Непрерывная обработка очереди

Scheduler loop обрабатывает пачки pending звонков без 10-минутных пауз между ними. Раньше: 10 звонков → пауза 10 мин → 10 звонков. Теперь: 10 → 10 → 10 → ... → пауза (только когда очередь пуста).
